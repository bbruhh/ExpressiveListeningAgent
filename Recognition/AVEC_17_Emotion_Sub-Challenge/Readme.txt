Readme - AVEC 17 Emotion Sub-Challenge

Data structure:

audio/: Audio data (.wav) 
 - unprocessed, 48 kHz

video/: Video data (.avi) 
 - unprocessed, 50 fps

transcriptions/: Manual transcriptions of the speech of the corresponding subject
 - format [start time; stop time; 'transcription'] (for each speaker turn)
 - includes also nonverbal vocalisations in brackets <>, such as <laughter>, <filler>, <breathing>, etc.
 - Inaudible utterances are marked as '...'

turns/: Turn timings (.csv)
 - format [start time; stop time] (for each speaker turn)

audio_features/: Audio features (.csv) 
 - 23 LLDs from the eGeMAPSv0.1a feature set (extracted with openSMILE, see full list below)
 - hopsize: 10 ms -> 100 frames per second

audio_features_functionals_4(6)s/: Audio features (.csv) 
 - 88 acoustic features from the eGeMAPSv0.1a feature set (extracted with openSMILE, see full list below)
 - block size: 4 (6 seconds)
 - hopsize: 100 ms -> 10 feature vectors per second, matches the sample rate of the labels

audio_features_xbow_4(6)s/: Bag-of-audio-words representation of the LLDs
 - Generated by openXBOW, options: -attributes nt1[23] -size 1000 -a 1 -standardizeInput -writeName -writeTimeStamp -log -t 4.0/6.0 0.1
 - block size: 4 (6 seconds)
 - hopsize: 100 ms -> 10 feature vectors per second, matches the sample rate of the labels

video_features/: Video features (.csv)
 - 1. face orientation (.csv) - in terms of pitch, yaw and roll (in degrees)
 - 2. pixel coordinates for 10 eye points (.csv)
 - 3. pixel coordinates for 49 facial landmarks (.csv)
 - format [instancename; timestamp; pitch; yaw; roll; x1; y1; x2; y2; ... x10; y10; x1; y1; x2; y2; ... x49; y49], (-1) if face is not detected
 - hopsize: 20ms -> sampled from 50 fps

video_features_normalised/: Video features (.csv)
 - Same as video_features/, but the x / y coordinates for eye points and the x / y coordinates of facial landmarks have been normalised (standardised) to zero mean and unit standard deviation on frame level

video_features_xbow_4(6)s/: Bag-of-video-words representation of the normalised video features
 - Generated by openXBOW, options: -attributes nt1[3]3[20]4[98] -size 1000 -a 1 -standardizeInput -writeName -writeTimeStamp -log -t 4.0/6.0 0.1
 - block size: 4 (6 seconds)
 - hopsize: 100 ms -> 10 feature vectors per second, matches the sample rate of the labels

labels/: Emotion labels (.csv)
 - arousal, valence, & liking
 - format [instancename;timestamp;arousal;valence;liking]
 - hopsize: 100 ms -> 10 ratings per second

Metadata.xlsx: Table with metadata
